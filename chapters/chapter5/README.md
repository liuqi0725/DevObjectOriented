## 第五章 何时使用面向对象编程

### 怎么识别一个对象

把对象当"对象"来对待。同时包含"数据"、"行为"的可以视之为对象。

### 论"数据"与"行为"

很难去界定该用对象还是方法。显然直接用方法的方式要比对象的方式代码更少，更紧凑。
但是代码少，不是衡量代码复杂程度的量。因该从可读性，可维护性上来进行区分。
使用对象的情况因该从以下等几个方面来考虑：

+ 复杂的数据
+ 有多种特定操作逻辑
+ 与其他模块有大量交互
+ ...

### 使用 Property 添加行为 

python 不像 java ，属性是开放的。没有私有变量，所以不像 java class 属性有 get，set 方法。
但是在特定的环境下，我们希望 get 或 set 属性时，能做一些逻辑处理。这时我们可以自己创建 get，set 方法。
